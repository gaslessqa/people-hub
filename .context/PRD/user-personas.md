Mujer de 28 anos, profesional, en una oficina moderna de startup. Tiene laptop con multiples pestanas abiertas, post-its en el monitor, y una taza de cafe. Expresion concentrada pero ligeramente frustrada.

### Demographics

| Atributo | Valor |
|----------|-------|
| **Edad** | 28 anos |
| **Ocupacion** | Talent Acquisition Specialist |
| **Ubicacion** | Ciudad de Mexico, Mexico |
| **Nivel educativo** | Licenciatura en Psicologia Organizacional |
| **Empresa** | Startup fintech de 45 empleados |

### Goals

1. **Cerrar vacantes rapidamente** sin sacrificar calidad de candidatos
2. **Mantener pipeline organizado** para saber exactamente donde esta cada candidato
3. **Reducir tiempo administrativo** para enfocarse en entrevistas y sourcing
4. **Tener informacion accesible** cuando managers preguntan por candidatos

1. **Spreadsheet nightmare:** Tiene 3 hojas de calculo para tracking, ninguna actualizada al 100%
2. **Buscar historial es un infierno:** Para verificar si alguien ya fue contactado, debe buscar en email, WhatsApp, LinkedIn y spreadsheets
3. **Feedback perdido:** Managers le dan feedback por Slack que luego no encuentra
4. **Contexto en transiciones:** Cuando contrata a alguien, tiene que volver a pedir informacion que ya tenia

### Tech Savviness

| Aspecto | Detalle |
|---------|---------|
| **Adopcion** | Early adopter - prueba nuevas herramientas constantemente |
| **Dispositivos** | MacBook Pro (trabajo), iPhone (personal) |
| **Apps frecuentes** | LinkedIn Recruiter, Slack, Google Sheets, Notion, WhatsApp |
| **Frustracion tech** | "Todas prometen integrar todo, ninguna lo hace bien" |

### Quote Representativa

> "Paso mas tiempo actualizando spreadsheets y buscando emails que haciendo mi trabajo real: conectar con candidatos y entender que necesita cada equipo."

### Goals

3. **Asegurar compliance** en procesos de contratacion

## Persona 4: Diego - El Super Admin

### Foto Description
Hombre de 30 anos, estilo casual tech (jeans, camiseta de conferencia tech), multiples monitores con consolas y dashboards. Expresion de "resolviendo problemas".

### Demographics

| Atributo | Valor |
|----------|-------|
| **Edad** | 30 anos |
| **Ocupacion** | IT Manager / DevOps |
| **Ubicacion** | Santiago, Chile |
| **Nivel educativo** | Ingenieria en Computacion |
| **Empresa** | Startup de 50 empleados |

### Goals

1. **Seguridad de datos** - informacion sensible de empleados/candidatos protegida
2. **Gestion de accesos** - que cada rol vea solo lo que debe ver
3. **Mantenimiento minimo** - sistema que no requiera intervencion constante
4. **Integracion con SSO** (futuro) - un login para todas las herramientas

### Pain Points

1. **Permisos ad-hoc:** Cada herramienta tiene su propio sistema de permisos
2. **Shadow IT:** RRHH usa spreadsheets compartidas con permisos mal configurados
3. **Onboarding tedioso:** Dar de alta usuarios en cada sistema es manual
4. **Auditoria imposible:** No puede saber quien accedio a que informacion

### Tech Savviness

| Aspecto | Detalle |
|---------|---------|
| **Adopcion** | Early adopter tech - evalua nuevas herramientas por arquitectura |
| **Dispositivos** | Linux laptop (trabajo), multiple homelab |
| **Apps frecuentes** | Terminal, AWS Console, Slack, 1Password, GitHub |
| **Frustracion tech** | "Sistemas HR legacy con seguridad de los 90s" |

### Quote Representativa

> "Necesito poder auditar quien vio informacion sensible. Y si alguien se va de la empresa, revocar acceso en un click, no en 20 sistemas diferentes."
